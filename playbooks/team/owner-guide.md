# 프롬프트 오너 가이드

프롬프트 자산의 품질과 안정성을 책임지는 오너 제도입니다.

---

## 오너 제도 개요

```
┌─────────────────────────────────────────────────────────────┐
│                      오너 제도                               │
├─────────────────────────────────────────────────────────────┤
│  정의: 프롬프트 자산의 품질과 안정성을 책임지는 담당자       │
│  기준: "가장 많이 쓰는 사람"이 아닌                          │
│        "품질/사고를 책임질 수 있는 사람"                     │
└─────────────────────────────────────────────────────────────┘
```

---

## 오너 지정 원칙

### 지정 기준
- 프롬프트 자산(템플릿)마다 **Owner 1명 이상** 지정
- 백업 오너 지정 권장 (휴가/퇴사 대비)

### 적합한 오너
- 해당 도메인 전문 지식 보유
- 프롬프트 품질 기준 이해
- 변경 영향 범위 파악 가능
- 팀 내 의사결정 권한 보유

---

## 오너 책임

| 책임 영역 | 상세 내용 |
|----------|----------|
| **품질 기준 유지** | 출력 형식, 금칙, 안전 규칙 관리 |
| **PR 승인** | 주요 변경/정책 변경에 대한 최종 승인권 |
| **회귀 테스트 관리** | 새 실패 케이스 반영, 세트 유지보수 |
| **드리프트 감지** | 월 1회 최소 점검, 모델 업데이트 대응 |
| **사고 대응** | 잘못된 안내/정책 위반/PII 등 즉시 대응 |

## 오너 활동 주기

### 일상 활동

| 주기 | 활동 |
|------|------|
| 수시 | PR 리뷰 및 승인 |
| 수시 | 실패 케이스 회귀 세트 반영 |
| 수시 | 사고 발생 시 즉시 대응 |

### 정기 활동

| 주기 | 활동 |
|------|------|
| 주 1회 | 주요 프롬프트 동작 점검 |
| 월 1회 | 전체 회귀 세트 실행 |
| 분기 1회 | 프롬프트 품질 전면 리뷰 |
| 모델 업데이트 시 | 드리프트 체크 |

---

## 드리프트 감지

### 드리프트란?
모델 업데이트나 시간 경과에 따라 동일 프롬프트의 출력이 변화하는 현상

### 감지 방법

```markdown
## 드리프트 체크리스트

### 출력 형식
- [ ] JSON 키/구조 유지
- [ ] 필수 필드 포함
- [ ] 길이 제약 준수

### 품질
- [ ] 정확도 유지
- [ ] 일관성 유지
- [ ] 톤/스타일 유지

### 안전
- [ ] 인젝션 방어 유효
- [ ] 금칙어 준수
- [ ] 정책 준수
```

### 대응 방법

1. 드리프트 발견 시 회귀 세트에 추가
2. 필요 시 프롬프트 조정 PR
3. 변경 로그 기록

---

## 사고 대응

### 사고 유형

| 유형 | 심각도 | 대응 시간 |
|------|--------|----------|
| 잘못된 정보 제공 | 🔴 High | 즉시 |
| 정책 위반 | 🔴 High | 즉시 |
| PII 노출 | 🔴 Critical | 즉시 |
| 형식 깨짐 | 🟡 Medium | 4시간 내 |
| 품질 저하 | 🟢 Low | 24시간 내 |

### 대응 프로세스

```
1. 문제 인지 및 기록
   ↓
2. 영향 범위 파악
   ↓
3. 임시 조치 (필요 시 롤백)
   ↓
4. 근본 원인 분석
   ↓
5. 수정 및 회귀 테스트 추가
   ↓
6. 사후 분석 문서화
```

## 오너 인수인계

### 인수인계 체크리스트

```markdown
## 오너 인수인계

### 문서
- [ ] 프롬프트 명세서 전달
- [ ] 회귀 테스트 세트 위치
- [ ] 변경 로그 히스토리
- [ ] 과거 사고 기록

### 권한
- [ ] PR 승인 권한 이전
- [ ] 알림 채널 구독
- [ ] 관련 시스템 접근권

### 지식
- [ ] 도메인 맥락 설명
- [ ] 주요 제약 조건
- [ ] 과거 이슈 및 해결 방법
- [ ] 주요 연동 시스템

### 연습
- [ ] 회귀 테스트 1회 실행
- [ ] 간단한 PR 리뷰 1회
```
